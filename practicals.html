Лекції Практичні Тести Матеріали

Практичні роботи

Закріпіть теоретичні знання на практиці. Кожна практична робота відповідає матеріалу відповідних лекцій.

01

Аналіз моделі розподілу відповідальності

До лекцій 1-2: Архітектура NIST та стандартизація

Нова

Проаналізуйте модель Shared Responsibility для різних хмарних провайдерів та типів сервісів.

Завдання:

  1. Порівняйте матриці відповідальності AWS, Azure та GCP для IaaS/PaaS/SaaS
  2. Визначте зони відповідальності клієнта для конкретного сценарію
  3. Складіть чек-лист контролів безпеки згідно ISO/IEC 27017
  4. Проаналізуйте типові SLA хмарних провайдерів
02

Проєктування Zero Trust архітектури

До лекції 3: Архітектура нульової довіри

Скоро

Розробіть архітектуру Zero Trust для гіпотетичної організації з хмарною інфраструктурою.

Завдання:

  1. Визначте критичні ресурси та потоки даних
  2. Спроєктуйте мікросегментацію мережі
  3. Опишіть політики доступу (PEP/PDP)
  4. Запропонуйте механізми верифікації ідентичності
03

Безпека контейнерів та Kubernetes

До лекції 4: Безпека віртуалізації та контейнеризації

Скоро

Налаштуйте безпечне середовище Kubernetes та проведіть аудит конфігурації.

Завдання:

  1. Розгорніть тестовий Kubernetes кластер (minikube/kind)
  2. Налаштуйте RBAC політики та Network Policies
  3. Проскануйте образи контейнерів на вразливості (Trivy)
  4. Перевірте відповідність CIS Kubernetes Benchmark
04

Аудит конфігурації хмарних ресурсів (CSPM)

До лекцій 7-8: CSPM та CNAPP

Скоро

Використайте інструменти CSPM для виявлення помилок конфігурації в хмарному середовищі.

Завдання:

  1. Налаштуйте Scout Suite або Prowler для AWS/Azure
  2. Проведіть сканування тестового середовища
  3. Проаналізуйте знайдені вразливості за CIS Benchmarks
  4. Складіть план усунення критичних проблем
05

Шифрування та управління ключами

До лекції 10: Захист даних та криптографія

Скоро

Практичне налаштування шифрування даних та управління ключами в хмарі.

Завдання:

  1. Створіть та налаштуйте ключі в AWS KMS або Azure Key Vault
  2. Налаштуйте шифрування даних at rest для сховища
  3. Реалізуйте envelope encryption для додатку
  4. Налаштуйте ротацію ключів та аудит доступу
06

Аналіз реального інциденту безпеки

До лекції 11: Актуальні загрози та аналіз інцидентів

Скоро

Детальний аналіз реального інциденту хмарної безпеки та розробка рекомендацій.

Завдання:

  1. Оберіть інцидент з публічних звітів (Capital One, Snowflake тощо)
  2. Відтворіть хронологію атаки та вектори проникнення
  3. Визначте порушені контролі безпеки
  4. Розробіть план превентивних заходів
